<cftextinput>

<cftextinput>

  • Home
  • Reference
  • Lucee tags
  • <cftextinput>

Lets you place a single-line text entry box in a cfform. The tag also gives you control over font characteristics.

Usage

<cftextinput
    name="string"
    [value="string"]
    [required="boolean"]
    [range="string"]
    [validate="boolean"]
    [onValidate="string"]
    [message="string"]
    [onError="string"]
    [size="number"]
    [font="string"]
    [fontSize="number"]
    [italic="boolean"]
    [bold="boolean"]
    [height="number"]
    [width="number"]
    [vSpace="number"]
    [hSpace="number"]
    [align="string"]
    [bgColor="string"]
    [textColor="string"]
    [maxLength="number"]
    [notSupported="string"]
>

This tag cannot have a body.

This tag is also supported within cfscript

Attributes

Attribute Description
name
(string, required)

A name for the cftextinput control.
value
(string, optional)

Initial value that displays in the text control.
required
(boolean, optional)

Yes or No. If Yes, the user must enter or change text. Default is No.
range
(string, optional)

Minimum value, maximum value range, separated by a comma. Valid only for numeric data.
validate
(boolean, optional)

date: verifies format mm/dd/yy. eurodate: verifies date format dd/mm/yyyy. time: verifies time format hh:mm:ss. float: verifies floating point format. integer: verifies integer format. telephone: verifies telephone format ###-###-####. The separator can be a blank. Area code and exchange must begin with digit 1 - 9. zipcode: verifies, in U.S. formats only, 5- or 9-digit format #####-####. The separator can be a blank. creditcard: strips blanks and dashes; verifies number using mod10 algorithm. Number must have 13-16 digits. social_security_number: verifies format ###-##-####. The separator can be a blank. regular_expression: matches input against pattern attribute.
onValidate
(string, optional)

The name of a JavaScript function used to validate user input. The form object, input object, and input object value are passed to the routine, which should return TRUE if validation succeeds and FALSE otherwise. When used, the validate attribute is ignored.
message
(string, optional)

Message text to display if validation fails.
onError
(string, optional)

Name of a JavaScript function to execute in the event of a failed validation.
size
(number, optional)

Number of characters displayed before horizontal scroll bar displays.
font
(string, optional)

Font name for text.
fontSize
(number, optional)

Font size for text.
italic
(boolean, optional)

Yes for italicized text, No for normal text. Default is No.
bold
(boolean, optional)

Yes for boldface text, No for medium text. Default is No.
height
(number, optional)

Height value of the control, in pixels.
width
(number, optional)

Width value of the control, in pixels.
vSpace
(number, optional)

Vertical spacing of the control, in pixels.
hSpace
(number, optional)

Horizontal spacing of the control, in pixels.
align
(string, optional)

Alignment value.
bgColor
(string, optional)

Background color of the control.
textColor
(string, optional)

Text color for the control. See bgColor for color options.
maxLength
(number, optional)

The maximum length of text entered.
notSupported
(string, optional)

The text to display if the page containing a Java applet-based cfform control is opened by a browser that does not support Java or has Java support disabled.
