#Handle unquoted tag attribute values as strings# Unquoted values for tag attributes are handled as strings by default, however with Lucee 5 there is now a setting in the Administrator (under Settings - Language/Compiler) where you can select to handle these values as variables instead.

Take this example:

<cfmail subject=subject from=from to=to />

With the default setting, this is interpreted as strings, e.g.:

<cfmail subject="subject" from="from" to="to" />

However when this setting is enabled, it is interpreted as variables, e.g.:

<cfmail subject="#subject#" from="#from#" to="#to#" />
